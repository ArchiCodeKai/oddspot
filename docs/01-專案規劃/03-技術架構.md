# 技術架構

## 核心技術棧

| 層面 | 技術 | 版本 | 說明 |
|------|------|------|------|
| Framework | Next.js | 16.1.6 | App Router |
| Language | TypeScript | 5 | strict mode |
| Styling | Tailwind CSS | 4 | |
| State Management | Zustand | 5 | 輕量 global state |
| Data Fetching | TanStack React Query | 5 | Server state + cache |
| ORM | Prisma | 7.5.0 | PostgreSQL |
| Auth | NextAuth | 5 (beta.30) | OAuth (Google / GitHub) |
| i18n | next-intl | 4 | App Router 相容 |
| Animation | Framer Motion | 12 | 滑卡片動效 |
| Map | @vis.gl/react-google-maps | - | Google 官方 React 地圖 |
| React Compiler | babel-plugin-react-compiler | - | 自動 memo 優化 |

## 地圖套件選擇

使用 **@vis.gl/react-google-maps**（Google 官方維護）而非 @react-google-maps/api：
- App Router SSR 相容（無 hydration 問題）
- Google 官方維護，長期穩定
- TypeScript 支援完整

## 資料庫策略

- **開發環境**: PostgreSQL（透過 `DATABASE_URL`）
- **生產環境**: PostgreSQL（Neon / Vercel 環境變數）
- Prisma 7 需要使用 adapter 模式（`PrismaBetterSqlite3`）

## 圖片策略

- **v1 Seed data**: 本地靜態圖 `/public/spots/`（最簡單）
- **v2 用戶上傳**: Cloudinary（到訪驗證模組一起實作）
- 儲存方式：`images` 欄位存 JSON string 陣列，第一張 = coverImage

## 專案結構

```
src/
  app/              # Next.js App Router pages
    api/            # API routes
    map/            # 地圖頁
    spots/[id]/     # 景點詳情頁
  components/
    map/            # 地圖相關元件
    ui/             # 通用 UI 元件
  lib/
    constants/      # 分類、狀態、路由、半徑常數
    db.ts           # Prisma singleton
  services/         # API 呼叫封裝（待實作）
  store/            # Zustand stores（待實作）
  types/            # TypeScript 型別定義
```

## 套件使用建議（避免造輪子）

| 功能 | 建議套件 | 狀態 | 說明 |
|------|---------|------|------|
| 滑卡片動效 | `framer-motion` drag | ✅ 已安裝 | 調好 `dragElastic`、`dragTransition` 參數即可，不需換套件 |
| Toast 通知 | `sonner` | ⚠️ 建議引入 | Vercel 出品，Next.js App Router 原生支援，比自製 toast 穩定 |
| 地圖 | `@vis.gl/react-google-maps` | ✅ 已安裝 | Google 官方 React 封裝 |
| 資料快取 | `@tanstack/react-query` | ✅ 已安裝 | 未來 spots fetch 建議遷移至此 |
| 圖片上傳（v2） | `uploadthing` | ⏳ v2 | Next.js 最佳搭配，type-safe，免費額度充裕 |
| 表單（v2 評論） | `react-hook-form` | ⏳ v2 | 效能最佳，與 Zod 搭配驗證 |
| 路線最佳化 | 自製（Haversine + 最近鄰） | ⏳ v2 | n≤5 不需套件，50 行解決 |
| 日期格式化（v2） | `date-fns` | ⏳ v2 | 輕量 tree-shakeable，比 dayjs 更現代 |
| 手勢進階控制 | `@use-gesture/react` | ⏳ 視需要 | 若 framer-motion drag 仍不夠細緻再引入 |
| Auth | `next-auth` | ✅ 已安裝 | Step 5 實作 |

> **原則**：已安裝的套件優先用好，不要為了功能重疊再裝新套件。`sonner` 是目前唯一建議盡早引入的。
