# 資料模型設計

## Spot（核心景點表）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | String | CUID |
| name | String | 中文名稱 |
| nameEn | String? | 英文名稱 |
| description | String? | 中文描述 |
| descriptionEn | String? | 英文描述 |
| lat / lng | Float | 經緯度，有複合 index |
| address | String? | 地址（非必填）|
| category | String | 8 個分類之一 |
| status | String | active / uncertain / disappeared |
| difficulty | String | easy / medium / hard |
| images | String | **JSON string 陣列**，第一張 = coverImage |
| rating | Float | 預設 0，v1 不顯示 |
| visitCount | Int | 預設 0，v1 不更新 |
| lastVerifiedAt | DateTime? | 最後驗證時間，v2 使用 |
| recommendedTime | String? | 推薦到訪時段 |
| legend | String? | 傳說/來由（自由文字）|
| googlePlaceId | String? | Unique，seed data 用 `seed-xxx` 佔位 |

### 注意事項
- `images` 存 `'["url1","url2"]'`，需 `JSON.parse()` 取值
- `coverImage = JSON.parse(images)[0]`
- `rating` / `visitCount` 欄位存在但 v1 無更新機制

## SavedSpot（收藏紀錄）

userId + spotId 唯一複合鍵，防止重複收藏。

## VisitedSpot（到訪紀錄，v2）

目前結構簡單（userId + spotId + visitedAt）。
v2 加入：`photos String`（用戶上傳照片 JSON）、`note String?`

## User（使用者）

NextAuth 5 相容版本，包含：
- `emailVerified DateTime?`（NextAuth 必要）
- `accounts Account[]`（OAuth token 關聯）
- `sessions Session[]`（登入 session 關聯）

## Account / Session / VerificationToken

NextAuth 5 Prisma Adapter 標準 models，不可改名。
Step 5 實作認證時直接使用。
